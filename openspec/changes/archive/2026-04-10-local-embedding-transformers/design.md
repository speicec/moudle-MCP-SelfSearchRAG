## Context

当前系统架构中，文本和图片 embedding 通过服务实现，但存在两个关键问题：
1. **Mock 实现**：生成伪随机向量，无语义意义，导致检索质量极差
2. **缺少多模态**：图片 embedding 服务未实现，无法进行图文检索

用户需求：
- **中英文支持**：处理中文和英文混合文档
- **多模态图片识别**：PDF 中的图片可被检索
- **企业化方案**：健壮、可离线部署

**解决方案**：集成 transformers.js，使用多语言文本模型和 CLIP 多模态模型。

## Goals / Non-Goals

**Goals:**
- 实现本地文本 embedding，支持中英文等 100+ 语言
- 实现多模态 embedding，支持图文跨模态检索
- 零 API 成本，离线可用
- 保持现有 API 接口兼容
- 企业化：模型管理、错误处理、配置灵活

**Non-Goals:**
- 不改变分块策略或检索算法
- 不引入 GPU 加速（保持纯 CPU 运行）
- 不支持视频/音频模态（仅图片）

## Decisions

### 1. 文本模型选择

**决定**：使用 `Xenova/multilingual-e5-small`

**对比分析**：
```
┌─────────────────────────────────────────────────────────────────────┐
│                        文本模型对比                                  │
├─────────────────────────┬──────────┬──────────┬────────────────────┤
│ 模型                    │ 维度     │ 大小     │ 语言支持           │
├─────────────────────────┼──────────┼──────────┼────────────────────┤
│ all-MiniLM-L6-v2        │ 384      │ ~25MB    │ ❌ 仅英文          │
│ (原提案)                │          │          │                    │
├─────────────────────────┼──────────┼──────────┼────────────────────┤
│ multilingual-e5-small   │ 384      │ ~118MB   │ ✅ 100+语言含中英文│
│ (新选择)                │          │          │ 推荐 ✓            │
├─────────────────────────┼──────────┼──────────┼────────────────────┤
│ paraphrase-multilingual │ 384      │ ~420MB   │ ✅ 50+语言         │
│ -MiniLM-L12             │          │          │ 体积较大          │
├─────────────────────────┼──────────┼──────────┼────────────────────┤
│ bge-m3                  │ 1024     │ ~2.2GB   │ ✅ 最强但过大      │
│ (若有JS版)              │          │          │ 不适合本地部署    │
└─────────────────────────┴──────────┴──────────┴────────────────────┘
```

**实现**：
```typescript
import { pipeline } from '@xenova/transformers';

const textEmbedder = await pipeline(
  'feature-extraction',
  'Xenova/multilingual-e5-small',
  { quantized: true }
);
```

### 4. 配置策略

**决定**：通过环境变量配置，支持灵活切换

```bash
# 文本 Embedding 模式
EMBEDDING_MODE=local                    # local | api

# 本地模型配置
LOCAL_TEXT_MODEL=multilingual-e5-small  # 文本模型
LOCAL_MULTIMODAL_MODEL=clip-vit-base-patch32  # 多模态模型

# API 模式配置（可选）
EMBEDDING_API_KEY=xxx
EMBEDDING_API_BASE_URL=https://api.openai.com/v1
```

### 5. 模型缓存与离线部署

**决定**：使用 transformers.js 内置缓存 + 支持预下载

```typescript
// 缓存位置
// Windows: %USERPROFILE%\.cache\huggingface\hub\
// macOS/Linux: ~/.cache/huggingface/hub/

// 预下载命令
npm run download-models  // 启动前预下载所有模型
```

## Risks / Trade-offs

### 1. 模型体积与下载

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 首次下载时间长 | 用户体验差 | 显示进度条，支持预下载 |
| 网络不稳定 | 下载失败 | 断点续传，重试机制 |
| 磁盘空间 | ~500MB | 清晰提示，支持自定义缓存路径 |

### 2. 内存与性能

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 模型加载内存占用 | ~400MB | 延迟加载，闲置卸载 |
| CPU 推理延迟 | 20-50ms/text | 批量处理，预热缓存 |
| 并发请求压力 | 内存暴涨 | 请求队列，并发限制 |

### 3. 多模态复杂度

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 图片处理慢 | 50-200ms/image | 异步处理，结果缓存 |
| 图片格式多样 | 解析失败 | 格式检测，统一转换 |
| 向量维度不同 | 384 vs 512 | 分开存储，按需检索 |

## Migration Plan

1. **安装依赖**：`npm install @xenova/transformers`
2. **创建服务**：
   - `LocalTextEmbeddingService` (multilingual-e5-small)
   - `MultimodalEmbeddingService` (clip-vit-base-patch32)
3. **工厂模式**：`EmbeddingServiceFactory`
4. **配置集成**：更新 `http-server.ts`
5. **预下载脚本**：`npm run download-models`
6. **测试验证**：中英文检索 + 图文跨模态
