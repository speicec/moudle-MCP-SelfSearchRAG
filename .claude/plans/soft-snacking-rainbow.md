# RAG MCP Server 开发计划

## 项目上下文

**目标**：创建一个基于 Harness 工程范式的 RAG MCP Server，采用 spec 文档驱动开发。

**核心理念**：`Model + Harness = Agent`
- 人类工程师从"写代码"转向"设计环境"
- 为 LLM Agent 构建完整运行环境：工具集、约束规则、反馈回路、上下文管理、观测审计、流程编排

## 技术选型

| 维度 | 选择 |
|------|------|
| 语言/运行时 | TypeScript + Node.js |
| MCP SDK | @modelcontextprotocol/sdk |
| 向量数据库 | Milvus |
| Embedding | 自定义/本地（支持可配置） |
| LLM | 自定义/本地（支持可配置） |
| 检索模式 | 混合检索（向量+全文+代码） |

## 开发阶段

### Phase 1: Spec 文档与项目初始化
1. 创建项目结构
2. 编写 spec 文档：
   - `specs/architecture.md` - 架构规格
   - `specs/tools.md` - MCP Tools 规格
   - `specs/resources.md` - MCP Resources 规格
   - `specs/prompts.md` - MCP Prompts 规格
   - `specs/harness.md` - Harness 配置规格
3. 初始化 package.json、tsconfig.json 等

### Phase 2: 核心基础设施
1. Milvus 连接与向量操作
2. 文档处理管道（分块、embedding）
3. 元数据存储（SQLite）

### Phase 3: RAG 引擎
1. 向量检索实现
2. 全文检索实现
3. 代码检索实现
4. 混合检索融合器

### Phase 4: Harness 层
1. 工具集定义
2. 约束规则引擎
3. 反馈回路机制
4. 上下文管理器
5. 观测审计系统
6. 流程编排器

### Phase 5: MCP Server 层
1. MCP Tools 暴露
2. MCP Resources 暴露
3. MCP Prompts 暴露
4. 服务器启动与配置

### Phase 6: 测试与文档
1. 单元测试
2. 集成测试
3. README 与使用文档
